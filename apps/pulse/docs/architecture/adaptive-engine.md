# Adaptive Engine & Image System

Status: canonical


## Content Style DNA (`src/intelligence/content-dna.ts`)

The brain of the adaptive system. Replaces ALL hardcoded type guidance. Tracks four signal types:

| Signal | Source | What it learns |
|---|---|---|
| Approval | User publishes a post | Avg length, question usage, numbers, emoji, first person |
| Rejection | User deletes a draft | Patterns to avoid (rhetorical openers, hype words, too long) |
| Edit | User modifies before publishing | Conciseness pref, question removal, data addition, tone shifts |
| Engagement | Post metrics after 4-36h | Best length range, top/bottom themes, best posting hours |

`buildDNAGuidance()` returns learned guidance injected into every content prompt:
- <3 posts: empty (zero assumptions)
- 3+ approved: "aim for ~180 chars, include specific numbers"
- 3+ edits: "be more concise — user frequently shortens"
- 10+ with engagement: "high-engagement topics: x402, circuit breakers"

## How Content Generation Works

1. Pick topic from content themes (knowledge notes / auto-research)
2. Pick angle from DNA angles (or random if no angles yet)
3. Build prompt: brand identity + niche knowledge + DNA guidance + topic
4. LLM generates with brand context — no category labels, no hardcoded instructions
5. Post validated, humanized, published
6. Outcome tracked back into DNA

## Content Rules (user-visible, editable)

8 default integrity rules stored in `brand-profile.contentRules`. Visible in Settings with toggle/edit/delete/add. Enforced in every content prompt via `buildRulesBlock()`.

## Brand Stance (auto-derived)

Auto-derived from brand identity. Prevents the bot from arguing against its own product. User-refinable through chat.

## LLM-Powered Topic Discovery (`src/intelligence/topic-discovery.ts`)

Search topics generated by LLM from brand knowledge. 8 X/Twitter search queries per discovery. ~0.5 credits per discovery.

## Other Adaptive Features

- **Engagement scoring weights** — per-brand, learned after 15+ posts
- **Dynamic billing** — actual token usage × 1.15 markup
- **Brand profile auto-migration** — backfills missing fields on every load
- **Brand profile auto-population** — extracts from knowledge notes when empty
- **Autopost DNA integration** — scheduler injects DNA/rules/stance into every prompt
- **X Free tier handling** — likes instead of replies to strangers, draft with intent URLs
- **LLM-powered relevance filter** — two-pass intent + relevance scoring
- **Persisted reply drafts** — saved to content_queue, loaded on Autopilot page

## Image System

Images are a first-class feature — upload, generate, auto-attach to posts.

**Pure tag matching**: `findBestImage()` scores on tag overlap (+40/tag, +50 for >50% match) > starred (+100) > less-used (+30/(count+1)). Per-image adaptive cooldowns.

**Asset library** (`src/core/asset-library.ts`): JSON metadata in `data/assets.json` + files in `data/assets/`.

**Image generation** (`src/intelligence/image-gen.ts`): Via ClawNet — Flux Schnell (~3cr), Flux Pro (~15cr), Freepik Mystic (~12cr).

**X media upload** (`src/platforms/x.ts`): v2 API (`api.x.com/2/media/upload`).

**Autopilot image mode** (`imageMode` in pulse.yaml): `auto` | `library` | `off` (default: library).

**API endpoints**: GET/POST `/api/media/*`, POST `/api/media/generate`, POST `/api/media/:id/caption`.
